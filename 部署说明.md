# GitHub Pages 部署说明

## 部署步骤

### 1. 初始化Git仓库（如果还没有）

```bash
cd 大数据导论/习题测验
git init
```

### 2. 添加远程仓库

```bash
git remote add origin https://github.com/Zhijun622/big_data_test.git
```

或者如果已经存在其他远程仓库，先删除再添加：

```bash
git remote remove origin
git remote add origin https://github.com/Zhijun622/big_data_test.git
```

### 3. 添加所有文件

```bash
git add .
```

### 4. 提交更改

```bash
git commit -m "初始提交：大数据导论选择题练习系统"
```

### 5. 推送到GitHub

```bash
git branch -M main
git push -u origin main
```

### 6. 启用GitHub Pages

1. 访问 https://github.com/Zhijun622/big_data_test
2. 点击 Settings（设置）
3. 在左侧菜单找到 Pages
4. 在 Source 部分选择：
   - Source: GitHub Actions
5. 保存设置

### 7. 等待自动部署

- GitHub Actions 会自动构建和部署
- 部署完成后，网站地址为：https://zhijun622.github.io/big_data_test/

## 自动部署

项目已配置 GitHub Actions 工作流，每次推送到 `main` 分支时会自动：
1. 安装依赖
2. 构建项目
3. 部署到 GitHub Pages

## 本地测试构建

在推送之前，可以本地测试构建：

```bash
npm run build
npm run preview
```

## 注意事项

1. **Base路径**：已配置为 `/big_data_test/`，与仓库名称一致
2. **路由**：使用 `BrowserRouter` 的 `basename` 属性支持子路径
3. **静态资源**：`questions.json` 已放在 `public` 目录，会自动复制到构建输出

## 更新部署

每次修改代码后：

```bash
git add .
git commit -m "更新：描述你的更改"
git push
```

GitHub Actions 会自动重新部署。
